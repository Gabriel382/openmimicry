# Contributing

## Development setup

```bash
cp .env.example .env
make install PROFILE=basic
make doctor
make run
```

## Branch naming

Use descriptive branch names such as:
- `feat/...`
- `fix/...`
- `docs/...`
- `chore/...`

## Commit style

Prefer conventional commits:
- `feat:`
- `fix:`
- `docs:`
- `chore:`
- `refactor:`
- `test:`

## Pull requests

- keep PRs focused
- explain what changed and why
- include screenshots or terminal output when useful
- update docs when behavior changes
